## 1. Objective & Business Outcome
`validate_backlog.py` and `verify_workflow.py` surface two non-blocking advisories: (1) the per-cycle `Wiki-Impact: none` rate, and (2) the per-item freshness lag between `manifest.items[id].updated_at_commit` and HEAD. Neither blocks the gate; both report to stderr for the agent or human to judge.

## 2. In-Scope Requirements
- [ ] New function in `scripts/validate_backlog.py`: `waiver_advisory(repo_root, cycle_id) -> list[str]`. Scans the cycle's B-items under `docs/megaplan/backlog-items/<cycle>-B*.md`, parses each metadata table, counts `Wiki-Impact: none` vs total. Returns an advisory string in the format `"<cycle>: <n>/<m> B-items waived Wiki-Impact (<pct>%) — confirm cycle scope is consistent"`.
- [ ] New function in `scripts/validate_backlog.py`: `freshness_advisory(repo_root) -> list[str]`. Reads `_meta/manifest.json` (if the wiki exists), computes per-item lag via `git rev-list --count <sha>..HEAD` for each `updated_at_commit`. Returns advisory strings in the format `"<id>: <n> commits behind HEAD (recorded <short-sha>)"`.
- [ ] Wire both into `validate_backlog.py`'s main entry. Print advisories to stderr; do not raise or exit non-zero.
- [ ] In `scripts/verify_workflow.py` `check_step_gate` for `document-post`, call `freshness_advisory` for the current item; if non-empty, print to stderr (advisory, not error).
- [ ] Tests in `tests/test_validate_backlog.py`: rate computation, freshness computation, advisory text format, no-blocking behavior, no-wiki opt-in (no errors when `wiki/` absent), no-manifest edge case (empty items dict).
- [ ] ADR: `docs/megaplan/adr/ADR-001-wiki-advisories-are-non-blocking.md` documenting why the rate and freshness checks are advisories, not gates. Three-condition ADR check: (1) hard to reverse — yes, gating would change agent behavior; (2) surprising without context — yes, blocking on waivers would force agents to populate the wiki; (3) real trade-off — yes, between strictness and the "derived/disposable" stance.
