## Outcome

At the `document (post) → COMPLETE` transition, `verify_workflow.py check` reminds the agent when the B-item has touched files but the wiki manifest entry is missing or stale. The reminder is non-blocking — the agent can either patch the wiki and re-ingest, or confirm `Wiki-Impact: none` was set intentionally. Structural validation errors (manifest corruption, frontmatter failure) remain blocking because they indicate real bugs.

- [ ] In `scripts/verify_workflow.py` `check_layer3_wiki`, replace the `errors.append(...)` calls for the missing-ingestion-record case (line ~300) and the stale-ingestion case (line ~308) with `print(f"  ! Layer 3 advisory (wiki): ...", file=sys.stderr)` calls. The text of the advisory should be specific and actionable.

- No change to the manifest schema.
