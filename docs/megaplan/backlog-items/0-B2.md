## Outcome

`ingest_wiki.py` no longer writes `_meta/ingestion.log`. The data is already in `_meta/manifest.json` (per-item ID, commit, file count); the log was redundant, polluting diffs, and recoverable from `git log wiki/_meta/manifest.json` when history is needed.

## Scope

- [ ] Remove the `log_path = ...` block from `ingest_wiki.py` (the append-to-log at the end of `ingest()`).
- [ ] Update the docstring's "Ingestion is split into a deterministic half and an authored half" note in `docs/methodology.md` to remove the now-irrelevant ingestion.log mention (there is none currently, but verify).
- [ ] New test in `tests/test_ingest_wiki.py`: after running `ingest()` on a fixture repo, `_meta/ingestion.log` is NOT created.

## Non-goals

- No change to `touched_files` computation.
- No gitignore rule (the file is simply not written).
