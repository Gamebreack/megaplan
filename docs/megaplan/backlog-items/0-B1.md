## Outcome

When an agent runs `python scripts/ingest_wiki.py <b_item>`, the manifest at `wiki/_meta/manifest.json` now carries a `suggested_pages` field â€” a deterministic list of `(wiki_relpath, [h2_anchors])` tuples the agent can use as a starting point for the authored `pages[]` decision.

## Scope

- [ ] New module `scripts/_wiki_map.py` with `suggest_pages(repo_root, touched_files) -> list[tuple[str, list[str]]]`.
- [ ] Module-slug derivation rules: `src/<x>/...` â†’ `<x>`; `tests/` or `test_` prefix stripped; root-level files use parent dir; `pkg/internal/<x>` â†’ `<x>`; etc.
- [ ] Page-matching: `architecture/<slug>.md` first, then `notes/<slug>.md`, then `contracts/<api>.md` only for files matching the contract heuristic (filename contains `route|endpoint|api|schema|graphql|openapi` OR path contains `routes/`, `api/`, `endpoints/`, `schema/`).
- [ ] Heading extraction: parse the matched page's H2 headings and return them all as the second element of the tuple.
- [ ] Integration: `ingest_wiki.py` calls `suggest_pages` and writes the result as `manifest.items[<id>].suggested_pages` (list of two-element lists, since tuples don't JSON-serialize).
- [ ] New test file `tests/test_ingest_wiki.py` with at least: slug derivation (one test per rule), page matching, heading extraction, empty touched_files, no match, multiple matches, manifest `suggested_pages` written.
- [ ] `validate_wiki.py` advisory: when `touched_files` non-empty, `suggested_pages` non-empty, `pages` empty â†’ print `! <id>: manifest.suggested_pages is non-empty but pages[] is empty â€” confirm Wiki-Impact: none or list patched pages`. Non-blocking.

## Non-goals

- No change to the authored `pages` field's semantics.
- No change to `touched_files` computation.
- No change to the contract-page matching beyond filename/path heuristics (no AST, no import-graph).
- No machine-learning-based or LLM-based file-to-page mapping.

## Dependencies / blockers

- None. Builds on the Layer 3 machinery already committed in `b865ff8`.
