# ADR-001: Use Prisma as ORM

## Status

Accepted

## Context

The project needs a database access layer for PostgreSQL. Options considered: Prisma, TypeORM, raw `pg` driver.

## Decision

Use Prisma ORM.

## Rationale

- Type-safe query builder with auto-generated types from schema
- Simple migration workflow via `prisma migrate`
- Better DX than TypeORM for this project size; raw driver adds too much boilerplate

## Consequences

- Schema lives in `prisma/schema.prisma` — migrations must be run before the app starts
- Prisma Client must be regenerated after schema changes (`npx prisma generate`)
