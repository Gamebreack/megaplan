## Outcome

Team had a working Todo API backed by an in-process `Map<string, Todo>` to validate the route shape before committing to a database.

## Scope

- [x] In-memory `TodoStore` with get/set/delete/list
- [x] Wire the store into the route handlers
- [x] Demo seed on server boot
- [x] Smoke test confirming routes work end-to-end

## Non-goals

- Persistence across restarts (the whole point of this prototype was to defer that decision)
- Concurrency safety (single-process, single-instance only)
